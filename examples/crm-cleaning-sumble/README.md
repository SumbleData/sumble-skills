# Example: CRM cleaning — Sumble's own CRM

A real, runnable output of the
[`sumble-crm-cleaning`](../../skills/sumble-crm-cleaning) skill, produced by
running it against Sumble's own Salesforce (1,236 accounts, ~10 minutes,
~5 API credits per account).

```bash
python3 app.py
# open http://localhost:8002
```

No `pip install`, no dependencies — Python 3.10+ and a browser.

## What you're looking at

- **Duplicates** — 12 clusters of CRM accounts that resolve to the *same*
  Sumble organization (the only duplicate evidence the skill uses — no fuzzy
  name matching). Includes pairs no name matcher would catch: Synopsys +
  BlackDuck (acquisition), Wordpress VIP + A8c (both Automattic — hover the
  "+N" chip on the match line to see the alternate names/domains that explain
  it). Each cluster shows the CRM footprint (contacts, opps, activities) per
  record and suggests a merge survivor.
- **Hierarchy gaps** — 17 missing parent links the org graph knows about
  (GitHub → Microsoft, Wiz → Google, HashiCorp → IBM, …) with the ancestor
  chain shown. A couple (Boomi → Dell) reflect since-divested ownership —
  exactly what the accept/reject review is for.
- **Parents not in CRM** — 27 parent companies missing entirely, grouped with
  the child accounts already present.
- **Unmatched** — 4 accounts Sumble couldn't resolve; usually shells or typos.

Accept/reject/skip each finding (decisions persist to `decisions.json`), pick
merge survivors, then **Export actions.csv** for the change list.

## What's real, what's fictitious

The accounts, duplicate clusters, hierarchy findings, and Sumble matches are
real output from the run. The **owner names and customer flags are
fictitious** (regenerated by `sanitize_findings.py`), and the CRM record
links point into a private Salesforce, so they won't resolve for you.

## Build this against your own CRM

Install the skill and run it — the interview takes a few minutes:

```bash
npx skills add SumbleData/sumble-skills --skill sumble-crm-cleaning
```

The method is written up in
[Clean your CRM against an org graph](../../skills/sumble-crm-cleaning/articles/01-clean-your-crm-against-the-org-graph.md).
